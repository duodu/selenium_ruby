selenium自动化框架优缺点
==================================================================
相较于现有“自动化测试平台”

优势
1. 对测试人员要求低（无须关心接口，只须关心业务以及测试）
2. 接近系统测试（能够验证所有页面元素及控件）
缺点
1. 脚本需要维护（页面和业务发生变化，相关脚本需要作出调整）（可以通过良好的设计减少这种工作量）
2. 效率可能低于“自动化测试平台”（调用浏览器，效率低于调用接口）
3. 只针对WEB项目（受限于selenium-webdriver的功能）

------------------------------------------------------------------
相较于手工测试

优势
1. 效率
2. 准确性
3. 可维护性（用例，缺陷）
缺点
1. 不能覆盖所有功能
2. 不能完全替代手工

------------------------------------------------------------------

selenium自动化实施方案
==================================================================
作用：
	用于验证基础的主要的业务流程
范围：
	在回归测试中进行
分工：
	功能测试人员：负责编写测试用例（输入，预期输出）
	非功能测试人员：负责实现被测业务流程的测试脚本

------------------------------------------------------------------

selenium自动化框架设计
==================================================================
设计模式：
	采用流行的page-object设计模式，即一个页面为一个对象，所有的元素检查，控件方法都为此对象的方法；
	测试用例，测试业务，对象识别，断言相分离，互相独立，便于维护；
	集成其他开源软件，做到用例管理，用例执行，到生成报告的全部功能。
